Wrangle Report

This report is a summary of the actions taken in the wrangle_act.ipynb file existing in this same folder and is designed to offer a general idea of the actions taken to gather, assess and clean data for this project. For any specific detail, please report to the above mentioned file.

Gathering:

We have worked with three different files:

  • Twitter_archive_enhanced.csv. A table containing 2356 tweets provided to us by Udacity.
  • images_predictions.tsv. A file containing the results of a image recognition algorithm applied to the list of tweets. We downloaded it using the Requests library.
  • tweet_json.txt. A txt file containing retweet, favorite count and urls from the Tweets in the twitter_archive_enhanced file stored in JSON format.  We downloaded it from Twitter leveraging the Tweepy library to access Twitter’s API.

Assessing:

After ensuring that all three tables were downloaded in our computer and stored in Dataframes in Jupyter, we proceed to visually and programmatically asses all three Dataframes.

For all of them we performed a visual analysis of the general structure and sample observations with the .head() and .sample() methods and, when for cases where we needed a deeper inspections —i.e. inspecting the texts columns in some observations— downloading the needed data into an excel with the to_csv() method.

Then we proceeded to understand the structure of the file (data types, missing values, summary statistics, frequencies, etc) programmatically with methods like .info(), .value_counts(), .describe(), .duplicated(), .isnull(), and we tried as well to get a better idea of the composition of single columns (i.e. finding denominators different to 10 or looking for rows with more than one dog category.

For ensuring a good organization, we documented all found issues and organized them into Tidiness and Quality Issues.

Cleaning:

Cleaning was the most challenging part of the wrangling process.  We had to meet the requirements of the assignment and ensure a good organization of the database while searching for documentation and  in Stackoverflow in order to meet the technical aspectS of the project.

We did not address all issues founded in the assessing, as the project requirements say that only 10 issues need to be addressed. As for our analysis, we are mostly using the clean_twitter_df we will focus our cleaning efforts  to that dataframe.

We will not detail all the procedures followed in the cleaning, which can be consulted at wrangle_act.ipynb, however we would like to emphasize a few things:

Good practice suggests dealing first with completeness matters, then with tidiness problems and finally cleaning up the rest of the quality issues.In our case, since we had to eliminate a not inconsiderable number of rows (Retweets, answers, tweets without photos, etc) we decided to deal with this first and then continue with the rest of the topics.  

Important to mention is that we had to make some decisions regarding WeRateDogs' categories of dogs:

  •  We added the category to several tweets that didn't have it based on the appearance of the term in the tweet text.

  • We added a new "mixed" category for tweets with several categories (mostly pictures with more than one dog)

  • Since we consider these categories to be part of a variable, we collapse the 4 existing columns into a "dog_type" column.
